Exercice 1 : Document HTML de base

  1. Que manque-t-il comme information à la balise <head> pour que le fichier s'affiche correctement ?

    C'est la Balise <meta> qui manque pour l'encodage (charset) : <meta charset="UTF-8">

  2. A quoi servent les tags h1, h2 et h3 ?

    Les tags h1, h2 et h3 sont des balises de titre utilisées pour définir l'importance et la hiérarchie du texte sur une page web. Plus le numéro dans la balise est élevé, plus le titre est considéré comme étant de niveau inférieur.

  3. A quoi sert le tag p ?

    Le tag <p> en HTML est utilisé pour définir des paragraphes dans une page web, permettant une structuration claire du contenu textuel.

  4. Les tags HTML :

    Les tags HTML commencent par une balise ouvrante <foo> et finissent par une balise fermante </foo>. Ces balises définissent la structure hiérarchique du contenu sur la page web.

  5. Comment mettre en gras ou en italique ?

    Pour mettre du texte en gras en HTML, on utilise la balise <b> : Gras

    Pour mettre du texte en italique en HTML, on utilise la balise <i> : Italique

  6. Comment faire pour mettre un mot à la fois en gras et en italique ?

    On utilise <b><i></i></b> pour mettre un mot à la fois en gras et en italique.

  7. Ajouter un titre à la fenêtre du navigateur

    On utilise la balise <title> avec le texte "Compte Rendu de TP1" à l'intérieur de la balise <head>

Exercice 2 : Uniform Resource Locator

  1. Quel est le nom de la machine ? Quel est le nom de domaine ?

    Concernant l'URL : http://monge.univ-mlv.fr/ens/ :

    Le nom de la machine : monge.univ-mlv.fr

    "monge.univ-mlv.fr" est un sous-domaine du domaine univ-mlv.fr Le nom de domaine complet comprend le sous-domaine "monge" suivi du domaine principal "univ-mlv.fr".

  2. Si on execute host monge.univ-mlv.fr

    La première indique l'adresse IPv4 associée au nom de l'hôte "monge.univ-mlv.fr" : "193.55.61.150"

    La seconde ligne indique que le courriel pour "monge.univ-mlv.fr" est géré par le serveur lui-même. Le MX (mail exchanger) pour "monge.univ-mlv.fr" a une priorité de 10 : "monge.univ-mlv.fr mail is handled by 10 monge.univ-mlv.fr."

  3. Pourquoi utilise-t-on des noms pour les machines plutôt que des adresses IP

    Les noms sont plus simples à utiliser à mémoriser pour les utilisateurs que les adresses IP numériques, qui sont difficiles à retenir.

  4. Que se passe-t-il si on essaye d'accéder à la page du cours dans le browser en remplaçant le nom de la machine par l'adresse IP trouvée précédemment ?

    La commande "host forax.github.io" liste 4 adresses IPv4 et 4 adresses IPv6.

    C'est une configuration de répartition de charge afin d'améliorer la performance.

    L'accès direct à une adresse IP pour une page GitHub (c'est le cas pour le cours) entraîne une erreur 404 car le serveur web s'attend à un en-tête Host spécifique, généralement fourni par le nom de domaine, pour router correctement la requête vers le site approprié.

  5. Qu'affiche la commande traceroute ?

    Elle affiche une liste de sauts (hops) avec les délais de réponse pour chaque saut.

    Certains sauts peuvent afficher "Délai d'attente de la demande dépassé", ce qui indique que la machine ne répond pas.

    Quel est le nom du routeur de sortie ?

    C'est l'adresse IPv6 2a01:e0a:d1d:5ac0::1

    Le nom n'est pas affiché, mais il peut être associé au fournisseur d'accès internet.






Exercice 3 : Service HTTP

  1. Question 1 :

    La commande "man services" affiche les informations du manuel associées au fichier /etc/services. Ce fichier est une liste qui fournit une correspondance entre les noms textuels des services Internet, leurs numéros de port assignés, et les types de protocoles associés.

  2. Question 2 :

    Le service HTTP est associé au port TCP 80 dans le fichier /etc/services. Le numéro 80 est le port standard utilisé pour les connexions HTTP.

  3. Question 3 :

    echo "GET / HTTP/0.9\r\n\r\n" | nc www.example.com 80

  4. Question 4 :

    HTTP/1.0 : C'est la version du protocole HTTP utilisée

    404 : C'est le code de statut de la réponse HTTP. "404" indique que la ressource demandée n'a pas été trouvée sur le serveur.

    Not Found : C'est la signification textuelle du code de statut

  5. Question 5 :

    Content-Type: text/html

    text/html indique que le contenu de la réponse est au format texte et qu'il est interprétable en tant que HTML.

  6. Question 6 :

    Content-Length: 345

    Signification : Le corps de la réponse a une longueur de 345 octets. Cela permet au client de connaître la taille du contenu avant de le recevoir complètement.

  7. Question 7 :

    echo "GET / HTTP/1.1\r\nHost: www.example.com\r\n\r\n" | nc www.example.com 80

    HTTP/1.1 200 OK

    HTTP/1.1 : C'est la version du protocole Hbody { font-family: 'Arial', sans-serif; margin: 20px; background-color: #f4f4f4; } h1, h2 { color: #333; text-align: center; } ol { list-style-type: decimal; padding-left: 20px; } p { line-height: 1.6; color: #555; } a { color: #ff7360; text-decoration: none; transition: color 0.3s ease; } a:hover { color: #79e2c3; text-decoration: underline; } img { max-width: 100%; height: auto; display: block; margin: 20px auto; border-radius: 8px; box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1); } code de statut.


    Content-Type: text/html; charset=UTF-8

    Elle indique que le contenu de la réponse est au format texte HTML et utilise l'encodage UTF-8.


    Content-Length: 1256

    La réponse est de longueur : 1256 octets.

  8. Question 8 :

    Dans la version 1.1 du protocole HTTP, l'inclusion de l'en-tête Host est obligatoire car un serveur peut héberger plusieurs sites web distincts sur la même adresse IP. Elle permet au serveur de savoir quel site web spécifique on souhaite accéder, ce qui est crucial pour les serveurs hébergeant plusieurs domaines sur la même adresse IP.

  9. Question 9 :

    echo "GET / HTTP/1.1\r\nHost: www.forax.org\r\n\r\n" | nc www.forax.org 80

    La réponse indique un statut de "302 Found", ce qui signifie que la ressource demandée a été temporairement déplacée vers une autre URI.

    Content-Type: text/html : Le type de média du contenu est du texte HTML.

    Transfer-Encoding: chunked : La réponse est encodée en morceaux

    La valeur de Content-Length n'est pas présente. Cela est souvent le cas lorsqu'une réponse utilise l'encodage "chunked", où le contenu est envoyé en morceaux plutôt qu'en une seule pièce.

    La valeur de Location indique l'URI vers laquelle la ressource demandée a été temporairement déplacée.
